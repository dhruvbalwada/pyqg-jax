Lagrangian Particles

Lagrangian particle advection on the model grid.

This module provides tools to advect passive Lagrangian particles using a gridded velocity field, as in pyqg.LagrangianParticleArray2D and pyqg.GriddedLagrangianParticleArray2D.

The velocities are assumed to live at cell centers (an Arakawa “A” grid), matching the real-space velocities produced by the models in this package. Interpolation is bilinear and the particles are advanced with the same two-time-level Runge-Kutta scheme used by PyQG.

Everything here is written in pure JAX, so the advection is differentiable, jax.jit()-compatible, and can be mapped over ensembles of particle sets with jax.vmap().

Added in version 0.9.0.

class pyqg_jax.particles.GriddedParticleStepper(*, L, W=None, periodic_in_x=True, periodic_in_y=True)[source]

Advect Lagrangian particles using a gridded velocity field.

The velocity fields are assumed to be defined at cell centers on a regular grid spanning \([0, L) \times [0, W)\), matching the real-space velocities of the models in this package. Interpolation to the particle positions is bilinear.

Parameters:

  • L (float) – Domain length in the x direction. Units: \(\mathrm{m}\).

  • W (float, optional) – Domain length in the y direction. Defaults to L. Units: \(\mathrm{m}\).

  • periodic_in_x (bool, optional) – Whether the domain wraps in the x direction. Defaults to True.

  • periodic_in_y (bool, optional) – Whether the domain wraps in the y direction. Defaults to True.

L, W

Domain lengths.

step(state, u0, v0, u1, v1, dt)[source]

Advance particles one step with a two-time-level RK4 scheme.

This matches the time stepping of pyqg.GriddedLagrangianParticleArray2D.step_forward_with_gridded_uv(): the first Runge-Kutta stage uses the velocity at the current time, and the remaining three stages use the velocity at the next time.

Parameters:

  • state (ParticleState) – The current particle positions.

  • u0 (jax.Array) – Gridded velocity components at the current time, shape (ny, nx).

  • v0 (jax.Array) – Gridded velocity components at the current time, shape (ny, nx).

  • u1 (jax.Array) – Gridded velocity components at the next time (a step dt later), shape (ny, nx).

  • v1 (jax.Array) – Gridded velocity components at the next time (a step dt later), shape (ny, nx).

  • dt (float) – Time step. Units: \(\mathrm{sec}\).

Returns:

The advected particle positions, a new object.

Return type:

ParticleState
