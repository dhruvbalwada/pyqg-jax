Layered QG Model

An implementation of pyqg.LayeredModel.

class pyqg_jax.layered_model.LayeredModel(*, nx=64, ny=None, nz=2, L=1000000.0, W=None, rek=5.787e-07, filterfac=23.6, f=None, g=9.81, beta=1.5e-11, rd=15000.0, delta=0.25, U=None, H=None, rho=None, precision=Precision.SINGLE)[source]

Layered quasi-geostrophic model with general number of layers.

See also pyqg.LayeredModel.

Added in version 0.9.0.

The potential vorticity anomalies \(q_i\) relate to the streamfunctions \(\psi_i\) through a tridiagonal stretching matrix \(\mathsf{S}\),

\[\hat{q} = (\mathsf{S} - \kappa^2 \mathsf{I})\,\hat\psi,\]

where \(\kappa^2\) is the horizontal wavenumber magnitude squared. The stretching matrix is built from the layer thicknesses \(H_i\) and the reduced gravities \(g'_i = g (\rho_{i+1} - \rho_i) / \rho_i\). The meridional background PV gradient is \(\mathsf{Q}_y = \beta - \mathsf{S}\,\mathsf{U}\).

Parameters:

  • nx (int, optional) – Number of grid points in the x direction.

  • ny (int, optional) – Number of grid points in the y direction. Defaults to nx.

  • nz (int, optional) – Number of layers (must be >= 2).

  • L (float, optional) – Domain length in the x direction. Units: \(\mathrm{m}\).

  • W (float, optional) – Domain length in the y direction. Defaults to L. Units: \(\mathrm{m}\).

  • rek (float, optional) – Linear drag in the bottom layer. Units: \(\mathrm{sec}^{-1}\).

  • filterfac (float, optional) – Amplitude of the spectral spherical filter.

  • f (float, optional) – Coriolis parameter. Units: \(\mathrm{sec}^{-1}\).

  • g (float, optional) – Acceleration due to gravity. Units: \(\mathrm{m}\ \mathrm{sec}^{-2}\).

  • beta (float, optional) – Gradient of the Coriolis parameter. Units: \(\mathrm{m}^{-1}\ \mathrm{sec}^{-1}\).

  • rd (float, optional) – Deformation radius. Units: \(\mathrm{m}\). Only used in the two-layer (nz == 2) case.

  • delta (float, optional) – Layer thickness ratio \(H_1 / H_2\). Unitless. Only used in the two-layer (nz == 2) case.

  • U (array of size nz, optional) – Background zonal velocity in each layer. Units: \(\mathrm{m}\ \mathrm{sec}^{-1}\).

  • H (array of size nz, optional) – Layer thicknesses. Units: \(\mathrm{m}\).

  • rho (array of size nz, optional) – Layer densities. Units: \(\mathrm{kg}\ \mathrm{m}^{-3}\). Used for nz >= 3 to build the stretching matrix.

  • precision (Precision, optional) – Precision of model computation. Selects dtype of state values.

Ubg[source]

The background velocity for this model.

Type:

jax.Array

Note

Only a zonal background flow is supported (no meridional background V), matching the pyqg-jax pseudo-spectral kernel, which carries only the \(\mathrm{i} k \mathsf{Q}_y\) advection term.

This model internally uses 64-bit floating point values for the inversion regardless of the chosen precision. Make sure that JAX has 64-bit precision enabled.

stability_analysis()[source]

Linear stability analysis of the background state.

Solves, at every wavenumber, the eigenvalue problem for normal modes \(\hat\theta \propto e^{-\mathrm{i}\omega t}\) of the linearized dynamics,

\[k\,(\mathsf{U} + \mathsf{Q}_y\,\mathsf{a})\,\tilde\theta = \omega\,\tilde\theta,\]

where \(\mathsf{U} = \mathrm{diag}(\mathtt{Ubg})\), \(\mathsf{Q}_y = \mathrm{diag}(\mathtt{Qy})\), and \(\mathsf{a}\) is the model’s inversion matrix (\(\hat\psi = \mathsf{a}\,\hat q\)). The growth rate at each wavenumber is \(\mathrm{Im}(\omega)\).

Added in version 0.9.0.

Returns:

  • omega (jax.Array) – The eigenvalue with the largest growth rate (imaginary part) at each wavenumber, shape (nl, nk). The growth rate is omega.imag and the phase speed is omega.real / k.

  • evec (jax.Array) – The corresponding eigenvector (the vertical structure of the most unstable mode) at each wavenumber, shape (nz, nl, nk).

Note

The eigenvalue problem is solved on the host with NumPy, so this is a host-side diagnostic (not differentiable or jit-compatible). It works regardless of the active JAX platform.
